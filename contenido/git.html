Git & GitHub

El Sistema de control de versiones moderno más utilizado en el mundo hoy es Git. Git es un proyecto de código abierto maduro y de mantenimiento activo desarrollado originalmente en 2005 por Linus Torvalds, el famoso creador del núcleo del sistema operativo Linux. Una asombrosa cantidad de proyectos de software dependen de Git para el control de versiones, incluidos proyectos comerciales y de código abierto.

¿Qué es Git y por qué deberíamos aprenderlo? Git es un VCS - Sistema de control de versiones. Lo que eso realmente significa es que Git nos ayuda a administrar nuestros archivos de proyecto. Una de las cosas principales que hace git y también la razón principal por la que existe es hacer un seguimiento de toda la historia de las cosas en las que está trabajando. Esto es especialmente útil para los desarrolladores de software porque cuando está trabajando en un proyecto, primero crea una versión básica del mismo y luego intenta mejorarlo agregando nuevas características (o) simplemente experimenta con cosas. Todo este proceso de experimentar con nuevas características es increíblemente propenso a errores y es posible que desee volver a su código original. Aquí es donde de control de versiones entra en juego, que sigue automáticamente todos los cambios minutos en su proyecto y nos permite volver a una versión anterior, no importa cuántas veces ha cambiado sus archivos. Otra cosa increíble que Git permite hacer es que permite a las personas trabajar juntas en el mismo proyecto al mismo tiempo sin alterar los archivos de los demás. La colaboración es mucho más fácil con Git. Los miembros del equipo pueden trabajar en diferentes funciones y combinar fácilmente los cambios.

GitHub es un servicio basado en la web para el control de versiones con Git. Básicamente, es un sitio de redes sociales para desarrolladores. Puede mirar el código de otras personas, identificar problemas con su código e incluso proponer cambios. Esto también te ayuda a mejorar tu código. En una nota más clara, es un gran lugar para mostrar sus proyectos y ser notado por posibles reclutadores. En resumen, Git es el sistema de control de versiones y GitHub es un servicio de alojamiento para repositorios Git.
